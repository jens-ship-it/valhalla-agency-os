Core Tables
Table Description Key Relationships
entity Organizations and vendors → contact_links, group_policies, service_tickets
contact Universal contact management → contact_links, service_tickets, deals, coi_records
contact_link Links contacts to entities contact ↔ entity
client Client-specific data for contacts → contact, individual_policies
user System users and staff → roles, service_tickets, deals
role User roles and permissions ↔ users (many-to-many)
service_ticket Service requests and issues → users, contacts, entities
group_policy Group insurance policies → entity
individual_policy Individual insurance policies → client
deal Sales opportunities → contact, entity, user
coi Center of Influence records → contact, user
vendor_detail Vendor-specific information → entity
distribution Broker-client relationships client_entity ↔ broker_entity
Key Design Patterns
  • Unified Entity Model: Organizations and vendors share the same base table with vendor-specific details in a separate table
  • Contact Links: Flexible many-to-many relationship between contacts and entities with role information
  • Polymorphic Service Tickets: Can be linked to either organizations or individual clients
  • Role-Based Access: Users have multiple roles that control system access
  • Client Hierarchy: Contacts can have optional client records for insurance-specific data
